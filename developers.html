Quick Start

Base URL

https://users.pdf-insights.ai

All API requests are sent to this base endpoint using HTTPS.

Try it in 30 seconds

Once you have an account and API key, upload a PDF and send a chat request. This is the core workflow external developers will use. 

curl -X POST https://users.pdf-insights.ai/pdf/upload \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "file=@manual.pdf"
curl -X POST https://users.pdf-insights.ai/chat \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "demo-session",
    "pdf_id": "YOUR_PDF_ID",
    "message": "What is the likely diagnosis?",
    "model": "gpt-4o-mini",
    "persona": "You are a senior refrigeration engineer."
  }'

Python Example

import requests

API_KEY = "YOUR_API_KEY"
BASE_URL = "https://users.pdf-insights.ai"

with open("manual.pdf", "rb") as f:
    upload = requests.post(
        f"{BASE_URL}/pdf/upload",
        headers={"Authorization": f"Bearer {API_KEY}"},
        files={"file": f}
    ).json()

result = requests.post(
    f"{BASE_URL}/chat",
    headers={"Authorization": f"Bearer {API_KEY}"},
    json={
        "session_id": "demo-session",
        "pdf_id": upload["pdf_id"],
        "message": "What is the likely diagnosis?",
        "model": "gpt-4o-mini",
        "persona": "You are a senior refrigeration engineer."
    }
).json()

print(result["answer"])

Core Endpoints

  • POST /pdf/upload — Upload and vectorize a PDF
  • POST /chat — Ask a question using a model and persona
  • GET /pdf/get/{pdf_id} — Retrieve a stored PDF

What makes the API different?

Most document APIs focus only on retrieval. PDF-Insights lets you shape the answer using a custom persona, so the same source document can be interpreted through the lens of a refrigeration engineer, safety inspector, legal reviewer, tutor, or rules official.

Use Cases

  • Industrial troubleshooting and technical manuals
  • Compliance and regulatory documentation
  • Legal and precedent-based document analysis
  • Training and internal knowledge assistants
  • Engineering reviews and service documentation
