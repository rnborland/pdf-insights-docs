PDF-Insights Developer API

Add persona-aware document reasoning to your applications in minutes.

Why PDF-Insights?

  • Persona-aware responses: Frame answers as an engineer, inspector, analyst, or rules official.
  • Document-grounded answers: Query PDFs without building your own RAG stack from scratch.
  • Usage and cost feedback: Receive answer, usage, and billing details in one flow.

Quick Start

  1. Create an account
  2. Generate an API key
  3. Upload a PDF
  4. Send a chat request with model and persona
  5. Receive an answer

Python Example

import requests

API_KEY = "YOUR_API_KEY"
BASE_URL = "https://users.pdf-insights.ai"

with open("manual.pdf", "rb") as f:
    upload = requests.post(
        f"{BASE_URL}/pdf/upload",
        headers={"Authorization": f"Bearer {API_KEY}"},
        files={"file": f}
    ).json()

result = requests.post(
    f"{BASE_URL}/chat",
    headers={"Authorization": f"Bearer {API_KEY}"},
    json={
        "session_id": "demo-session",
        "pdf_id": upload["pdf_id"],
        "message": "What is the likely diagnosis?",
        "model": "gpt-4o-mini",
        "persona": "You are a senior refrigeration engineer."
    }
).json()

print(result["answer"])

Core Endpoints

  • POST /pdf/upload — Upload and vectorize a PDF
  • POST /chat — Ask a question using a model and persona
  • GET /pdf/get/{pdf_id} — Retrieve a stored PDF

Authentication

Use bearer token authentication with your API key.

Authorization: Bearer YOUR_API_KEY

API keys are generated from your PDF-Insights account dashboard.
