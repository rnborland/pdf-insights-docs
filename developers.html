Developer API

Turn any PDF into a programmable AI assistant.

Upload a document, ask questions, apply a custom system prompt, and get structured responses with cost tracking.

Open PDF-Insights App View GitHub Docs

Who This Is For

This API is intended for developers who want to integrate PDF-Insights into their own applications, tools, or workflows.

Typical use cases include document assistants, internal knowledge tools, industrial manuals, compliance workflows, and training applications.

Getting Started

  1. Create an account using a username, email, and password.
  2. Log in and select Developer API (Get API Key).
  3. Generate and securely store your API key.
  4. Upload a PDF and save the returned pdf_id.
  5. Send questions to the /chat endpoint.

Authentication

API requests use Bearer authentication. 

Authorization: Bearer pdi_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

API keys begin with pdi_live_ and the raw key is shown once only, so store it securely.

Quick Start Example

The example below shows the complete flow: upload a PDF, then ask a question about it. 

import requests

API_KEY = "pdi_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
BASE_URL = "https://users.pdf-insights.ai"

headers = {
    "Authorization": f"Bearer {API_KEY}"
}

# 1. Upload PDF
with open("example.pdf", "rb") as f:
    upload_resp = requests.post(
        f"{BASE_URL}/pdf/upload",
        headers=headers,
        files={"file": ("example.pdf", f, "application/pdf")}
    )

upload_resp.raise_for_status()
pdf_id = upload_resp.json()["pdf_id"]

print("Uploaded PDF ID:", pdf_id)

# 2. Ask a question
chat_resp = requests.post(
    f"{BASE_URL}/chat",
    headers=headers,
    json={
        "pdf_id": pdf_id,
        "question": "Summarize this document",
        "system_prompt": "You are a helpful assistant"
    }
)

chat_resp.raise_for_status()
data = chat_resp.json()

print(data["answer"])
print(data["cost"])

Core Endpoints

POST /pdf/upload

Upload a PDF using multipart form data.

Response:

{
  "pdf_id": "your-pdf-id"
}

POST /chat

Ask a question about a previously uploaded PDF.

Request body:

{
  "pdf_id": "your-pdf-id",
  "question": "Summarize this document",
  "system_prompt": "You are a helpful assistant"
}

Example Response

Developer response:

{
  "answer": "Document summary here...",
  "status": "ok",
  "model_used": "gpt-4o-mini",
  "system_prompt_used": "You are a helpful assistant",
  "cost": {
    "charged_usd": 0.0021,
    "wallet_balance_usd": 4.9979
  }
}

Why system_prompt Matters

One of the key differentiators of PDF-Insights is that the same PDF can behave like different assistants depending on the system prompt you provide.

Examples:

  • You are a refrigeration engineer diagnosing failures.
  • You are a compliance auditor reviewing this document.
  • You are a safety trainer explaining this clearly.

This makes the API more than simple PDF search. It turns a document into a programmable assistant.
